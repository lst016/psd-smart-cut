# V2 基础原则

本文记录 `PSD Smart Cut V2` 已经确认下来的产品决策。

除非明确重新对齐，否则以下内容都视为锁定原则。

## 锁定决策

### 1. 主入口必须是 `AI orchestrator`

项目未来的主入口不是某个固定的 runtime pipeline，而是一个可以动态编排多 agent 的 orchestrator。

它负责：

- 看整页截图
- 理解页面与模块
- 拉起不同角色的 subagent
- 合并组件与资源计划
- 调用确定性工具执行导出

### 2. 页面理解必须 `截图优先`

页面和模块判断必须由截图或整页预览主导。

PSD 结构只能作为：

- 边界对齐的辅助
- 证据补充
- 歧义澄清

不能反过来由 PSD 结构主导产品判断。

### 3. 最终输出协议固定为四层

V2 的目标输出协议固定为：

- `pages/`
- `components/`
- `resources/`
- `analysis/`

含义如下：

- `pages/`：页面与模块实例，是主入口
- `components/`：可复用组件定义
- `resources/`：唯一物理资源
- `analysis/`：给人审阅和对齐的文档层

### 4. 资源策略默认 `复用优先`

系统应优先输出：

- 共享状态资源
- 可复用视觉皮肤
- 组件引用共享资源的关系

系统应尽量避免：

- 每个实例一张位图
- 视觉上相同的控件反复导出
- 明明可复用却被整块烘焙的 UI

典型例子：

- 按钮组：共享 `normal` / `selected` 或 `primary` / `secondary`
- switch：共享 `track_on` / `track_off` / `thumb`
- tab：共享状态底图，不为每个 tab 实例单独导图

### 5. 旧 pipeline 只保留为兼容层

当前 `heuristic pipeline` 继续保留，但只承担：

- 回归对照
- 兼容运行
- 实验辅助

它不再代表未来架构，也不应该继续无限扩张。

## V2 的产品定义

`PSD Smart Cut V2` 是一个多 agent AI 资源规划工具台，服务对象包括：

- Cursor
- Claude
- Codex

它帮助 agent：

- 看懂页面
- 拆清业务模块
- 定义前端意义上的组件
- 定义可复用美术资源
- 在计划收敛后执行确定性导出

## 核心工作流

```text
整页截图
-> 页面理解
-> 模块树
-> 区域树
-> 组件规划
-> 资源规划
-> 资源任务单
-> 并发切图
-> 复核
```

## V2 明确不做什么

V2 不应被定义为：

- 一个纯 PSD 结构解析器
- 一个按图层驱动的自动切图器
- 一个写死顺序的固定 pipeline
- 一个只给终端用户手点的 CLI 工具

## 这些原则意味着什么

一旦接受这套基础原则，就意味着：

1. 规划和导出必须彻底分离
2. 资源身份必须独立于组件实例
3. 组件设计必须来自角色判断，而不是启发式补丁
4. `analysis/` 必须持续可读、可审阅
5. orchestrator 必须拥有流程控制权
6. `resource-task-list` 和并发 worker 应成为正式工作流的一部分
