## 优先级判断

如果只做一件事，我建议先做：

1. 运行时与测试彻底解耦

原因：

- 风险最低
- 收益明确
- 能继续提升仓库对外可发布性

如果再做第二件事，我建议做：

2. 源码乱码清理

原因：

- 不一定影响功能
- 但会持续拖累维护、交接、审阅和后续重构效率
