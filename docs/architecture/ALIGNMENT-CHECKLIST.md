# 对齐清单

本文记录当前工作流里已经对齐和仍需继续对齐的关键点。

## 已经确认的部分

以下内容已经锁定：

1. 主入口是 `AI orchestrator`
2. 页面理解必须 `截图优先`
3. PSD 结构只做辅助证据
4. 最终输出协议是 `pages / components / resources / analysis`
5. 资源策略默认 `复用优先`
6. 旧 heuristic pipeline 只保留为兼容层
7. 切图必须先经过 `resource-task-list`

## 仍需持续对齐的部分

### 1. 页面、模块、子页面边界

需要持续确认：

- 一个 PSD 里的多个画面，到底是独立页面还是同一模块的子页面
- artboard 应映射到 page、module，还是只是视觉画布

当前样例已经确认：

- 3 个画面
- 2 个业务模块
- 其中 Android/iOS 是同一教学模块下的子页面

### 2. 角色职责边界

当前建议分工已经比较稳定：

- `product`：页面用途与模块边界
- `frontend`：组件模型与状态复用
- `art`：资源粒度与图片策略
- `export`：确定性导出
- `review`：最终审阅

仍需继续观察：

- 是否需要额外的 `qa` 或 `synthesizer` 角色

### 3. 资源粒度

这是当前最容易出问题的部分。

必须持续对齐：

- 哪些资源其实已经是可直接导出的标准层
- switch 是否永远优先抽成共享状态资源
- 按钮组是否永远优先抽成共享皮肤
- modal/header/body 应如何拆分
- 哪些大块背景应该整留，哪些应该拆开

### 4. 文本策略

必须持续对齐：

- 普通 UI 文案是否默认保持文本
- 广告强视觉文字何时必须烘焙进图
- 教学说明文字是做文本块配置还是做图片

### 5. 审阅机制

当前建议审阅顺序：

1. 页面理解
2. 模块树
3. 组件计划
4. 资源计划
5. 资源任务单
6. 导出结果

仍需继续确认：

- 是每阶段都人工确认
- 还是跑到 `review package` 再整体确认

### 6. 生产准入线

这一项本轮之后必须显式管理。

进入生产环境前，至少必须满足：

1. 资源导出策略已经按类型分流
2. 不再把所有资源走同一条导出链
3. 多层嵌套资源有正式的 `PSD 差分隔离` 方案
4. 单层图标、组合图标、状态资源、实例资源都有明确白名单策略
5. 抽样验证结果已经达到“前端和美术可直接使用”
6. 已建立“单层直出优先”的策略升级门槛

如果以上任一条件未满足，则只能视为：

- `试运行`
- `方法验证`
- `工程探索`

不能视为生产可用版本

## 当前工程优先级

如果按照现在的方向继续推进，工程优先级建议是：

1. 建立资源类型分流
2. 建立“单层直出探测”
3. 建立 `PSD 差分隔离`
4. 让 `resource-task-list` 真正驱动导出
5. 固化试运行包为回归样例
6. 继续把 orchestrator 输入输出协议落实成可执行格式
7. 把旧 pipeline 逐步下沉成工具层

## 每轮复盘时都该问的问题

1. 这轮是否先看懂了页面，再切资源
2. 这轮是否先拆清模块，再拆组件
3. 这轮是否优先抽了共享状态资源
4. 这轮是否把普通文案错误地烘焙成图
5. 这轮是否有重复导出的实例级图片
6. 这轮是否错误地把背景一起切进资源
7. 这轮是否错误地把互斥态叠进同一个资源
8. 这轮是否把本可直出的资源错误升级成复杂策略
9. 这轮资源是否已经达到生产可交付标准

## 本轮额外结论

根据最新大厅稿试运行：

1. `页面理解` 路径是正确的
2. `资源任务单` 路径是正确的
3. `通用视觉抠图` 不能作为主方案
4. 单层直出探测必须成为复杂策略前的必经步骤
5. 下一阶段必须转入 `资源类型分流 + PSD 差分隔离`
